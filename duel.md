# Эмодзи-дуэль — техническая документация

Эмодзи-дуэль — мини-игра. Двое участников чата сначала договариваются о поединке, затем соревнуются в нажатии кнопки с пистолетом. Исход каждого выстрела случаен: 60% попасть в соперника, 20% выстрелить себе, 20% промах.

---

## Запуск

### Команда `/duel`

Доступна только в группах (`ChatType.GROUPS`).

Бот отправляет сообщение с кнопками — по одной на каждого зарегистрированного участника чата, кроме вызвавшего. Кнопки расположены по две в ряд.

```
@caller, выбери соперника для дуэли:

[@vasya]  [@petya]
[@masha]
```

Состояние хранится в:
```
__pending_picks: dict[message_id → (chat_id, caller_id, caller_username, candidates)]
__pick_jobs:     dict[message_id → Job]  # таймер на 60 сек
```

Если зарегистрированных соперников нет — бот сообщает об этом и завершает работу.

### Выбор соперника

Только вызвавший может нажать кнопку. Зрителям — алерт «Это не твоя дуэль, зритель! 👀».

После выбора picker-сообщение редактируется:

```
⚔️ @caller вызывает @target на дуэль!
```

Затем бот отправляет отдельное challenge-сообщение с кнопками принятия/отклонения (фаза 1).

### Таймаут выбора (60 сек)

Если вызвавший не выбрал соперника, сообщение редактируется:

```
⏳ @caller не выбрал соперника. Время вышло.
```

## Фаза 1: вызов (ожидание согласия)

Бот отправляет новое сообщение с кнопками принятия/отклонения.

Бот отправляет одно сообщение с кнопками принятия/отклонения. Только p2 (тот, кого вызвали) может нажимать кнопки. У p2 есть **30 секунд** (`DUEL_ACCEPTANCE_TIMEOUT`).

```
⚔️ @player1 вызывает @player2 на дуэль!

@player2, принять вызов?

⏱ У @player2 есть 30 сек.

[✅ Принять]  [❌ Отклонить]
```

Состояние хранится в:
```
__pending_acceptance: dict[message_id → (chat_id, p1_id, p1_username, p2_id, p2_username)]
__acceptance_jobs:    dict[message_id → Job]  # таймер на 30 сек
```

### Защита от чужих нажатий

| Кто нажал | Кнопка              | Результат                                         |
|-----------|---------------------|---------------------------------------------------|
| p2        | Принять / Отклонить | Обрабатывается                                    |
| p1        | Принять             | Алерт «Ты уже бросил вызов, жди ответа!»         |
| p1        | Отклонить           | Алерт «Нельзя отклонить собственный вызов!»       |
| Зритель   | Любая               | Алерт «Это не твоя дуэль, зритель! 👀»           |

### p2 отклоняет

Сообщение редактируется, кнопки убираются. Примеры:

```
🏳️ @player2 отклонил вызов @player1. Трус.
❌ @player2 испугался и отказался от дуэли с @player1.
😂 @player2 убежал. @player1 остался стоять в одиночестве.
```

### p2 не отвечает (таймаут 30 сек)

Сообщение редактируется, кнопки убираются. Примеры:

```
⏳ @player2 не ответил на вызов @player1. Время вышло.
💨 Тишина. @player2 проигнорировал вызов @player1.
```

## Фаза 2: обратный отсчёт

После принятия запускается job с `run_once(..., delay=0)`. Он обновляет то же самое сообщение каждые 2 секунды, считая от 10 до 2:

```
⚔️ @player1 vs @player2

⏱ Дуэль начнётся через 10...
```
```
⚔️ @player1 vs @player2

⏱ Дуэль начнётся через 8...
```
...
```
⚔️ @player1 vs @player2

⏱ Дуэль начнётся через 2...
```

После последнего шага сообщение обновляется до активной дуэли с кнопкой (фаза 3).

---

## Фаза 3: активная дуэль (стрельба)

Бот редактирует сообщение, добавляя кнопку `🔫`. Кнопку может нажать любой из двух участников. Первый нажавший становится стрелком в этом раунде.

```
⚔️ @player1 vs @player2

[🔫]
```

Состояние хранится в:
```
__active_duels:      dict[message_id → (chat_id, p1_id, p1_username, p2_id, p2_username, shot_log: list[str])]
__duel_timeout_jobs: dict[message_id → Job]  # таймер на 300 сек
```

Callback data кнопки: `duel_fire`. Паттерн обработчика: `^duel_`.

### Защита от гонки

Asyncio однопоточен. `__handle_fire` первым делом делает `.pop()` из `__active_duels` — без каких-либо `await` до этого. Это гарантирует, что выстрел обработает только один обработчик. Зрителям дуэль немедленно возвращается обратно в `__active_duels`.

## Исходы выстрела

Результат определяется через `random.choices(["hit", "self", "miss"], weights=[60, 20, 20])`.

Каждая строка истории содержит время реакции стрелка — сколько прошло с момента появления кнопки до нажатия.

### Попадание в соперника (60%)

Стрелок побеждает. История выстрелов дополняется, кнопка убирается.

```
⚔️ @player1 vs @player2

💥 @player1 попал в @player2! Победитель: @player1! (⚡ 0.87 сек)
```

```
⚔️ @player1 vs @player2

🎯 Точный выстрел @player1 в @player2! @player1 победил! (⚡ 1.24 сек)
```

### Выстрел в себя (20%)

Стрелок проигрывает, соперник побеждает. История выстрелов дополняется, кнопка убирается.

```
⚔️ @player1 vs @player2

🦶 @player1 прострелил себе ногу. Дарвиновская премия почти в кармане. @player2 победил! (⚡ 0.43 сек)
```

Вариант с рикошетом:

```
⚔️ @player1 vs @player2

🔄 Пуля @player1 обогнула весь чат и вернулась к отправителю. @player2 даже не шелохнулся. (⚡ 2.11 сек)
```

### Промах (20%)

Дуэль продолжается. Строка промаха добавляется в историю, таймер реакции сбрасывается, кнопка `🔫` появляется снова.

```
⚔️ @player1 vs @player2

💨 @player1 промахнулся! (⚡ 0.61 сек)

[🔫]
```

Вариант с предохранителем:

```
⚔️ @player1 vs @player2

🔒 @player1 не снял предохранитель. Выстрела не было. (⚡ 0.38 сек)

[🔫]
```

### Несколько раундов

История накапливается в одном сообщении, каждый раунд со своим временем:

```
⚔️ @player1 vs @player2

💨 @player1 промахнулся! (⚡ 0.61 сек)
🌀 У @player2 дрогнула рука — промах! (⚡ 1.09 сек)
💨 @player1 промахнулся! (⚡ 0.44 сек)
💥 @player2 попал в @player1! Победитель: @player2! (⚡ 0.72 сек)
```

## Таймаут активной дуэли

Если никто не нажал кнопку в течение **5 минут** (`DUEL_FIRE_TIMEOUT = 300`), дуэль завершается. Текущая история сохраняется, к ней добавляется строка таймаута, кнопка убирается:

```
⚔️ @player1 vs @player2

💨 @player1 промахнулся!

⏳ Время вышло. Трусы.
```

Если выстрелов ещё не было:

```
⚔️ @player1 vs @player2

💨 Оба испугались курка.
```

## Статистика и достижения

При победе (попадание в соперника или соперник выстрелил себе) начисляется `duel_wins += 1` через `increment_stat`. Проигравшему ничего не начисляется.

| Порог | Ключ           | Название                |
|-------|----------------|-------------------------|
| 1     | `duel_win_1`   | Первая кровь            |
| 10    | `duel_win_10`  | Быстрый палец           |
| 25    | `duel_win_25`  | Гроза дикого чата       |
| 50    | `duel_win_50`  | Самый быстрый на диване |
| 100   | `duel_win_100` | Кибер-кликун            |
