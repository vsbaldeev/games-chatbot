# Эмодзи-дуэль — техническая документация

Эмодзи-дуэль — мини-игра на реакцию. Двое случайных участников чата получают одну кнопку с пистолетом. Кто нажмёт первым — тот победил.

---

## Запуск

### Команда `/duel`

Доступна только в группах (`ChatType.GROUPS`).

### Выбор участников

Берётся список участников из `chat_members`. Если в чате больше 2 участников, вызвавший команду исключается из пула. Двое выбираются случайно через `random.sample` — без повторений. Если в чате ровно 2 участника, исключение не применяется.

Если участников меньше 2 — бот сообщает об этом и завершает работу.

### Вызов через сообщение

Участник может вызвать конкретного человека, написав сообщение со словом «дуэл» (в любой форме) и @username цели. Упоминать бота **не обязательно**:

> хочу дуэль с @vasya
> вызываю @petya на дуэль

Бот ищет первый non-bot `@mention` среди entities сообщения и сопоставляет с участниками чата. Если совпадение найдено — запускает дуэль между отправителем и целью. Если не найден — сообщает об ошибке. Вызвать самого себя нельзя.

Если сообщение содержит «дуэл» и `@`, но цель не определена (нет non-bot mention), бот не реагирует и обрабатывает сообщение как обычное обращение.

### Сообщение с кнопкой

Бот отправляет одно сообщение с анонсом и инлайн-кнопкой:

```
⚔️ Эмодзи-дуэль!

@player1 против @player2

Первый нажмёт на курок — победит. Приготовьтесь...

[🔫]
```

Callback data кнопки: `duel_fire`.

Состояние дуэли сохраняется в памяти процесса:

```
__pending_duels: dict[message_id → (chat_id, p1_id, p1_username, p2_id, p2_username)]
```

## Обработка нажатия

При нажатии на кнопку срабатывает `CallbackQueryHandler` с паттерном `^duel_fire$`.

### Логика

1. Ищем дуэль по `message_id`. Если не найдена — показываем алерт «Дуэль уже завершена».
2. Проверяем, является ли нажавший одним из двух участников дуэли. Если нет — алерт «Это не твоя дуэль, зритель! 👀».
3. Удаляем дуэль из `__pending_duels` (атомарно через `.pop()` — до следующего `await`).
4. Редактируем исходное сообщение: убираем кнопку, выводим результат:

```
💥 @winner выстрелил первым!

@loser — слишком медленно. В следующий раз.
```

5. Победителю начисляется `duel_wins += 1`, проверяются достижения.

### Защита от гонки

Asyncio однопоточен. Обработчик первым делом делает `.pop()` из `__pending_duels` — без каких-либо `await` до этого. Это гарантирует, что дуэль заберёт только один обработчик, даже если оба участника нажали одновременно. Если нажавший оказался зрителем (не участником) — дуэль немедленно возвращается обратно в `__pending_duels`.

## Таймаут

Если никто не нажал кнопку в течение **5 минут** (`DUEL_TIMEOUT_SECONDS = 300`), дуэль завершается автоматически:

1. Кнопка убирается из сообщения (`edit_message_reply_markup` с `reply_markup=None`).
2. В чат отправляется сообщение с насмешкой над обоими участниками.

Примеры сообщений при таймауте:
```
⏳ Время вышло. @p1 и @p2 так и не решились нажать. Трусы.
💨 Дуэль @p1 vs @p2 отменяется — оба испугались курка.
🏳️ Никто не нажал. @p1 и @p2 разошлись по домам, поджав хвосты.
```

Таймер отменяется сразу при нажатии кнопки участником (`job.schedule_removal()`).

Таймер хранится в `__duel_jobs: dict[message_id → Job]`.

## Статистика и достижения

При победе начисляется `duel_wins += 1` через `increment_stat`. Проигравшему ничего не начисляется.

| Порог | Ключ | Название |
|-------|------|----------|
| 1 | `duel_win_1` | Первая кровь |
| 10 | `duel_win_10` | Быстрый палец |
| 25 | `duel_win_25` | Гроза дикого чата |
| 50 | `duel_win_50` | Самый быстрый на диване |
| 100 | `duel_win_100` | Кибер-кликун |
