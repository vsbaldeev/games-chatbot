# Система достижений — техническая документация

## Схема базы данных

### Таблица `user_stats`

Первичный ключ: `(user_id, chat_id)`

| Колонка | Тип | Описание |
|---------|-----|----------|
| `user_id` | INTEGER | Telegram ID пользователя |
| `chat_id` | INTEGER | Telegram ID чата |
| `username` | TEXT | Отображаемое имя |
| `laugh_reactions` | INTEGER | Полученные реакции 😂😁😆😀😃 |
| `heart_reactions` | INTEGER | Полученные реакции ❤️ (все цвета) |
| `fire_reactions` | INTEGER | Полученные реакции 🔥 |
| `thumbsup_reactions` | INTEGER | Полученные реакции 👍 |
| `emoji_messages` | INTEGER | Сообщения, содержащие хотя бы один эмодзи |
| `sticker_messages` | INTEGER | Отправленные стикеры |
| `forwarded_messages` | INTEGER | Пересланные текстовые сообщения |
| `link_messages` | INTEGER | Сообщения, содержащие URL |
| `voice_messages` | INTEGER | Голосовые сообщения (количество) |
| `video_messages` | INTEGER | Видеосообщения |
| `video_note_messages` | INTEGER | Кружочки (video note) |
| `photo_messages` | INTEGER | Фотографии (не пересланные) |
| `night_messages` | INTEGER | Сообщения в период 00:00–04:59 по МСК |
| `roasted_count` | INTEGER | Количество полученных прожарок |
| `roulette_win_count` | INTEGER | Количество выживаний в русской рулетке (осечка) |
| `duel_wins` | INTEGER | Количество побед в эмодзи-дуэлях |
| `long_messages` | INTEGER | Неиспользуемая колонка (легаси) |
| `voice_max_duration` | INTEGER | Длина самого длинного голосового, в секундах |
| `long_message_max` | INTEGER | Длина самого длинного текстового сообщения, в символах |
| `last_seen` | INTEGER | Unix-время последней активности пользователя в чате; используется для достижений молчания |

Все числовые колонки по умолчанию равны `0`. Строка создаётся при первом `increment_stat` или `update_max_stat` для пары `(user_id, chat_id)`.

При каждом вызове `increment_stat` или `update_max_stat` колонка `last_seen` обновляется через `strftime('%s','now')` прямо в теле SQL-апсёрта. Если значение `last_seen == 0`, считается, что активность у пользователя ещё не зафиксирована и достижения молчания не выдаются.

### Таблица `chat_members`

Первичный ключ: `(chat_id, user_id)`. Одна строка на каждого участника, которого бот когда-либо видел в чате.

### Таблица `announced_achievements`

Первичный ключ: `(user_id, chat_id, key)`. Хранит ключи уже объявленных достижений. Служит журналом идемпотентности — повторный вызов `check_new_achievements` не создаст дубликат уведомления.

### Таблица `message_authors`

Первичный ключ: `(chat_id, message_id)`.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `chat_id` | INTEGER | Telegram ID чата |
| `message_id` | INTEGER | Telegram ID сообщения внутри чата |
| `user_id` | INTEGER | Автор сообщения |
| `username` | TEXT | Отображаемое имя автора на момент записи |
| `created_at` | INTEGER | Unix-время вставки записи (для будущей чистки) |

Заполняется в `track_member` при каждом сообщении не от бота. Используется обработчиком реакций, чтобы найти автора, чей счётчик нужно увеличить, когда приходит обновление количества реакций. Запись переживает рестарт бота — реакции на старые сообщения больше не теряются из-за пустого in-memory кеша.

### Таблица `message_reaction_counts`

Первичный ключ: `(chat_id, message_id, emoji)`.

| Колонка | Тип | Описание |
|---------|-----|----------|
| `chat_id` | INTEGER | Telegram ID чата |
| `message_id` | INTEGER | Telegram ID сообщения |
| `emoji` | TEXT | Эмодзи реакции |
| `total_count` | INTEGER | Последнее известное количество этой реакции на сообщении |
| `updated_at` | INTEGER | Unix-время последнего обновления (для будущей чистки) |

Хранит снимок текущих счётчиков реакций по каждому сообщению. При получении нового `MessageReactionCountUpdated` обработчик сравнивает новое значение с сохранённым и засчитывает только положительную дельту. Удалённые реакции не вычитаются, но фиксируются в снимке, чтобы повторное добавление считалось заново.

## API обновления статистики

### `increment_stat(user_id, chat_id, username, stat)`

Увеличивает счётчик на 1. Использует `INSERT … ON CONFLICT DO UPDATE SET col = col + 1` — строка создаётся при первом вызове. Каждый вызов сбрасывает `last_seen = strftime('%s','now')`.

`stat` должен быть одним из `TRACKABLE_STATS`:
`laugh_reactions`, `heart_reactions`, `fire_reactions`, `thumbsup_reactions`,
`emoji_messages`, `sticker_messages`, `forwarded_messages`,
`link_messages`, `voice_messages`, `video_messages`, `video_note_messages`,
`photo_messages`, `night_messages`, `roasted_count`, `roulette_win_count`, `duel_wins`, `long_messages`

Передача неизвестного значения вызывает `ValueError`.

### `update_max_stat(user_id, chat_id, username, stat, value)`

Обновляет колонку только если `value` превышает текущее значение: `SET col = MAX(col, excluded.col)`. Каждый вызов сбрасывает `last_seen = strftime('%s','now')`.

`stat` должен быть одним из `MAX_TRACKABLE_STATS`: `voice_max_duration`, `long_message_max`

Передача неизвестного значения вызывает `ValueError`.

## Как собирается статистика

Каждый обработчик в `bot.py` обновляет статистику **до** любого раннего выхода (случайная вероятность ответа, кулдаун), поэтому каждое квалифицирующее событие всегда засчитывается.

### Текстовые сообщения — `handle_message`

Срабатывает на `filters.TEXT & ~filters.COMMAND` в группах.

| Условие | Стат |
|---------|------|
| Всегда | `night_messages` (только если 00:00–04:59 МСК) |
| Текст содержит эмодзи (U+1F300–1F9FF, U+2600–27FF, U+1FA00–1FAFF) | `emoji_messages` |
| Текст содержит `http://`, `https://` или `www.` | `link_messages` |
| У сообщения установлен `forward_origin` | `forwarded_messages` |
| Всегда | `update_max_stat("long_message_max", len(text))` |

### Голосовые и кружочки — `handle_voice_message`

Срабатывает на `filters.VOICE | filters.VIDEO_NOTE` в группах.

| Условие | Стат |
|---------|------|
| `msg.voice` присутствует | `voice_messages` |
| `msg.voice` присутствует | `update_max_stat("voice_max_duration", msg.voice.duration)` |
| `msg.video_note` присутствует | `video_note_messages` |

### Фото — `handle_photo_message`

Срабатывает на `filters.PHOTO` в группах. Пересланные фото и фото от бота пропускаются.

| Условие | Стат |
|---------|------|
| Всегда (не пересланное, не от бота) | `photo_messages` |

### Стикеры — `handle_sticker_message`

Срабатывает на `filters.Sticker.ALL` в группах. Стикеры от бота пропускаются.

| Условие | Стат |
|---------|------|
| Всегда | `sticker_messages` |

### Видео — `handle_video_message`

Срабатывает на `filters.VIDEO` в группах. Пересланные видео и видео от бота пропускаются.

| Условие | Стат |
|---------|------|
| Всегда (не пересланное, не от бота) | `video_messages` |

### Реакции — `handle_reaction_count`

Срабатывает на `MessageReactionCountUpdated`. Telegram присылает этот апдейт боту-неадмину агрегировано (с задержкой до ~1 минуты) и анонимно — без информации о том, кто именно поставил реакцию. Бот хранит снимок предыдущих счётчиков в `message_reaction_counts` и засчитывает только **положительную дельту** между новым и сохранённым значением. Удаление реакции не уменьшает накопленный стат.

Стат засчитывается **автору сообщения**, на которое поставили реакцию (берётся из `message_authors`). Если автор не найден в таблице (например, сообщение отправлено до того, как бот видел чат), апдейт молча игнорируется.

Чтобы Telegram действительно доставлял этот тип апдейтов, при запуске поллинга в `bot.py` явно указывается `allowed_updates=Update.ALL_TYPES` — по умолчанию `message_reaction_count` исключён.

| Эмодзи | Стат |
|--------|------|
| 😁 🤣 | `laugh_reactions` |
| ❤ 🥰 😍 💘 ❤️‍🔥 | `heart_reactions` |
| 🔥 | `fire_reactions` |
| 👍 | `thumbsup_reactions` |

Известные ограничения подсчёта по дельте:
- Сценарий «реакция поставлена → удалена → поставлена снова другим пользователем» создаёт ложноположительный засчёт (так как старое значение не откатывается, новое добавление выглядит как +1 поверх уже учтённой реакции). Для системы достижений с порогами 1/10/25/50/100 это пренебрежимо.
- При первом апдейте на сообщение, для которого нет записи в `message_reaction_counts`, дельтой считается полное текущее значение. Это корректно для свежих сообщений (бот видит автора и реакции с нуля); для старых сообщений, для которых нет записи в `message_authors`, апдейт не обрабатывается вовсе.

## Поток разблокировки достижений

### Достижения на основе статистики

После каждого обновления статистики `bot.py` вызывает:

```python
await __notify_unlocks(context, chat_id, user_id, username)
```

Функция выполняет:
1. `achievements.check_new_achievements(user_id, chat_id, username)` — вычисляет все заработанные достижения по текущей статистике, сравнивает с `announced_achievements`, вставляет новые ключи, возвращает только новые.
2. Для каждого нового достижения отправляет в групповой чат:
   ```
   🏆 @username получил достижение!

   {emoji} *{title}*
   _{description}_
   ```

Отправка использует `parse_mode="Markdown"` с plain-text запасным вариантом при `BadRequest` (например, у username есть спецсимволы). Все ошибки перехватываются и логируются как `WARNING` — сбой уведомления не прерывает основной обработчик.

`__notify_unlocks` вызывается из всех путей обновления статистики:
- После статистики текстовых сообщений (`handle_message`)
- После статистики голосовых и кружочков (`handle_voice_message`)
- После статистики фото (`handle_photo_message`)
- После статистики стикеров (`handle_sticker_message`)
- После статистики видео (`handle_video_message`)
- После статистики реакций (`handle_reaction_count`) — только если была положительная дельта
- После ответа бота (внутри `__send_agent_reply` и `__send_lightweight_reply`)

### Достижения молчания (time-based)

Достижения молчания не проверяются при каждом сообщении — они выдаются только фоновым заданием, которое ежедневно проходит по всем участникам всех чатов.

Ежедневно в 10:00 UTC (13:00 МСК) `__check_silence_achievements` в `bot.py`:
1. Получает все `chat_id` через `achievements.get_all_chat_ids()`
2. Для каждого чата — всех участников через `achievements.get_chat_members(chat_id)`
3. Вызывает `achievements.check_silence_achievements(user_id, chat_id, username)`
4. Отправляет уведомление в чат за каждое новое достижение (тот же формат, что и `__notify_unlocks`)

`check_silence_achievements` внутри:
1. Читает `last_seen` из `user_stats`; если `== 0` — выходит без результата
2. Вычисляет `elapsed_days = (time.time() - last_seen) / 86400`
3. Сопоставляет с `_SILENCE_THRESHOLDS` (3, 7, 14, 30 дней)
4. Возвращает только те достижения, которых ещё нет в `announced_achievements`, и сразу их помечает

Ключевое следствие: как только пользователь отправит любое сообщение, `last_seen` сбрасывается и счётчик молчания обнуляется. Уже выданные достижения молчания остаются — они помечены в `announced_achievements`.

## Команды

### `/achievements`

Показывает **последние 3 достижения** вызвавшего команду пользователя и общее количество заработанных. Формат каждого достижения:
```
{emoji} *{title}*
_{description}_
```
Заголовок: `Последние достижения {username} ({N} всего):`

Если достижений нет — саркастичное сообщение.

### `/top`

Показывает топ-3 участника чата по **количеству достижений** с медалями 🥇🥈🥉. Формат строки:
```
{medal} {username} — {N} достижений
```
Если ни у кого нет достижений — сообщение с подсказкой.

## Правила достижений

### `_ACHIEVEMENT_RULES` — на основе статистики

Каждая запись: `(колонка_статистики, пороговые_значения, ключи_достижений)`

Достижение засчитывается, когда значение статистики **≥ порогу**. Для `update_max_stat`-статов (`voice_max_duration`, `long_message_max`) сравнивается максимальное накопленное значение.

| Стат | Пороги | Ключи |
|------|--------|-------|
| `roasted_count` | 1, 10, 25, 50, 100 | `roasted_1` … `roasted_100` |
| `roulette_win_count` | 1, 5, 10, 20, 50 | `roulette_win_1` … `roulette_win_50` |
| `duel_wins` | 1, 10, 25, 50, 100 | `duel_win_1` … `duel_win_100` |
| `laugh_reactions` | 1, 10, 25, 50, 100 | `laugh_1` … `laugh_100` |
| `heart_reactions` | 1, 10, 25, 50, 100 | `heart_1` … `heart_100` |
| `fire_reactions` | 1, 10, 25, 50, 100 | `fire_1` … `fire_100` |
| `thumbsup_reactions` | 1, 10, 25, 50, 100 | `like_1` … `like_100` |
| `emoji_messages` | 1, 10, 25, 50, 100 | `emoji_1` … `emoji_100` |
| `sticker_messages` | 1, 10, 25, 50, 100 | `sticker_1` … `sticker_100` |
| `forwarded_messages` | 1, 10, 25, 50, 100 | `forward_1` … `forward_100` |
| `link_messages` | 1, 10, 25, 50, 100 | `link_1` … `link_100` |
| `voice_messages` | 1, 10, 25, 50, 100 | `voice_1` … `voice_100` |
| `voice_max_duration` | 60, 120, 180, 300, 600 (сек) | `voice_dur_1m` … `voice_dur_10m` |
| `video_messages` | 1, 10, 25, 50, 100 | `video_1` … `video_100` |
| `video_note_messages` | 1, 10, 25, 50, 100 | `circle_1` … `circle_100` |
| `photo_messages` | 1, 10, 25, 50, 100 | `photo_1` … `photo_100` |
| `night_messages` | 1, 10, 25, 50, 100 | `night_1` … `night_100` |
| `long_message_max` | 140, 250, 500, 1000, 2000 (симв) | `essay_140` … `essay_2000` |

### `_SILENCE_THRESHOLDS` — на основе времени

Сравнивается `(time.time() - last_seen) / 86400` с порогом в днях. Проверяется только фоновым заданием.

| Порог (дней) | Ключ |
|-------------|------|
| 3 | `silence_3d` |
| 7 | `silence_7d` |
| 14 | `silence_14d` |
| 30 | `silence_30d` |

Всего: **94 достижения** (90 на статистике + 4 на молчании).
